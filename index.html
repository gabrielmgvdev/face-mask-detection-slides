Detecção de Rostos em Pessoas com Máscaras Faciais

Processamento Digital de Imagens com OpenCV

Instituição

Centro Universitário Dom Helder

Stack

Python · OpenCV · Haar Cascades

1. O Problema

Contexto Sanitário

Necessidade crítica de monitoramento automatizado do uso de máscaras em grandes espaços públicos.

Objetivo Técnico

Criar um pipeline capaz de detectar rostos e identificar contornos (máscaras) mesmo em condições adversas.

O Dataset

Andrewmvd (Kaggle): 853 imagens variadas. Desafiador devido a iluminação e múltiplas faces por imagem.

Complexidade Visual

Multidões, oclusões e ângulos variados

2. Pipeline de Processamento

1. Pré-proc

  • • Conversão BGR ➔ Gray
  • • Gaussian Blur (5x5) para reduzir ruído de sensor
  • • Otsu Thresholding para análise inicial

2. Features

  • • Canny (80, 180): Superior ao Sobel para contornos finos.
  • • Detecção de bordas focada na transição Máscara/Pele.

3. Detecção

  • • Haar Cascade (Frontal Face)
  • • ScaleFactor: 1.1
  • • MinNeighbors: 4 (Equilíbrio Recall/Precision)

4. Discussão & Robustez

O Paradoxo do Haar

A hipótese inicial era de falha catastrófica devido à oclusão (máscara cobrindo boca/nariz). Contudo, o classificador manteve alta taxa de acerto.

Motivo: A região dos olhos, sobrancelhas e ponte nasal preserva contraste suficiente (características do tipo Haar) para ativação do classificador.

Limitações Encontradas

Rotação Facial

O modelo frontalface falha em rostos de perfil (>30 graus).

Iluminação Extrema

Sombras fortes criam "bordas falsas" no Canny, confundindo a análise da máscara.

Projeto Concluído

O pipeline desenvolvido atendeu a todos os requisitos do Capstone, entregando uma solução de monitoramento rápida (36 FPS) e funcional.
